About

About Leila

I am a computer science graduate student at NC State University, where I study how AI-assisted tools reshape software engineering workflows. My work combines empirical software engineering, machine learning, and large-scale data to build honest, practical tooling for developers.

My research sits at the intersection of software engineering and machine learning. I build models and tools that understand how developers and large language models co-author code, with the goal of improving reliability without hiding the messy reality of modern codebases.

Before graduate school, I worked on backend systems and developer tools in industry, from high-throughput inventory services to Kubernetes-based developer platforms. These experiences now shape a research agenda that is unapologetically grounded in real repositories, CI pipelines, and production constraints.

I aim to bridge empirical software engineering and practical tooling so that AI assistance in coding remains observable, reliable, and honest about its provenance.

SE-ADAPT: Defect Prediction for AI-Assisted Codebases
Ahmadi, L., Menzies, T. · ICSE 2026 · under review

Proposes an adaptive defect prediction framework that augments traditional software metrics with LLM provenance signals such as prompt entropy and generation churn, improving F1 over strong baselines on 12 open-source repositories.

Automated Technical Debt Localization Using Pre-trained Code Models
Ahmadi, L., Hosseini, R. · ICSME 2022 Workshop · 2022

Introduces a transformer-based approach for detecting and localizing technical debt in large Java codebases, achieving high precision on a 10K-method corpus.
