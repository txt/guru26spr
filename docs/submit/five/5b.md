# Smalltalk Homework Rubric

**Total: 10 points** | Extra credit: +2 pts

---

## Part A — Hello, messages, and printing · `hello.st` · 2 pts

| Criterion | Points |
|---|---|
| Code is present for printing a name, a number result, and a string size | 0.5 |
| `printNl` (or equivalent) called on at least 5 expressions | 0.5 |
| Variables declared and assigned for a number, a string, and an array literal | 0.5 |
| Print calls are labeled with descriptive string literals (not just bare values) | 0.5 |

---

## Part B — Collections and blocks · `collections.st` · 5 pts

| Criterion | Points |
|---|---|
| `do:` used with a block that sends `printNl` to each element | 0.75 |
| `collect:` used with a block that applies a transformation to each element | 1 |
| `select:` used with a block that tests a condition on each element | 1 |
| `reject:` used with a block that tests a condition on each element | 1 |
| `inject:into:` used with a block that accumulates a result | 1 |
| Each method call is preceded by a labeled string literal identifying it | 0.25 |

---

## Part C — Tiny DSL · `miniDsl.st` · 2 pts

| Criterion | Points |
|---|---|
| Comment block at top identifies the three-step repeated idiom | 0.25 |
| A helper method or class is defined with a clear, intent-expressing name | 0.5 |
| Helper body uses `select:`, `reject:`, and `inject:into:` (or equivalent) correctly | 0.75 |
| Helper is called on sample data with the result sent to `printNl` | 0.25 |
| Call site reads as a domain phrase rather than a sequence of collection operations | 0.25 |
